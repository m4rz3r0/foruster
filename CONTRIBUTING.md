# Contributing to Foruster

Thank you for your interest in contributing to Foruster. We genuinely appreciate the time and effort that users and professionals invest in improving the tool.

---

## Closed Source Policy

Foruster is **closed-source software** distributed under a proprietary End User License Agreement. As such:

- **We do not accept Pull Requests.** There is no public source code repository to contribute to, and submitting patches or modifications to the compiled binaries is strictly prohibited under the terms of the [LICENSE](./LICENSE).
- **Forks and derivative works are not permitted** without prior written authorization from the Foruster Team.

---

## How You Can Contribute

While code contributions are not possible, we strongly value the following forms of community participation:

### üêõ Bug Reports

If you encounter unexpected behavior, a crash, or an error, please open a detailed **Bug Report** using the issue template provided:

[‚Üí Open a Bug Report](.github/ISSUE_TEMPLATE/bug_report.md)

A high-quality bug report includes:
- Your operating system and Foruster version.
- A clear, step-by-step description of how to reproduce the issue.
- A description of the expected vs. actual behavior.
- Relevant screenshots or log excerpts (ensuring no sensitive data is included).

### üí° Feature Requests & Suggestions

We welcome suggestions for new features, workflow improvements, or integrations. Please open an issue with the title prefix `[FEATURE REQUEST]` and describe:

- The use case or forensic scenario you are addressing.
- The proposed functionality and its expected outcome.
- Any relevant references to forensic standards or best practices.

### üìñ Documentation Improvements

If you identify errors, ambiguities, or gaps in the documentation hosted in this repository, please open an issue with the title prefix `[DOCS]` and describe the correction or improvement you suggest.
